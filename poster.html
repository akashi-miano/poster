1. Abstract

Publicly available USB Rubber Ducky payloads are frequently detected and neutralized by modern endpoint defenses like Windows Defender, limiting their effectiveness in penetration testing and red team operations. This project addresses the critical challenge of creating undetectable, automated attack payloads by implementing advanced evasion techniques. A comprehensive Ducky Script was developed that leverages DLL Proxying to bypass real-time antivirus protection, UAC bypass for silent privilege escalation, and automated C2 agent deployment for persistent remote access. The payload successfully evades detection while demonstrating credential exfiltration through App-Bound Encryption bypass, providing a practical tool for security professionals to simulate realistic insider threats and validate organizational defenses.

2. Problem Statement & Objectives

Problem:
  • Publicly available Rubber Ducky payloads are outdated or easily neutralized by modern defenses.
  • Lack of comprehensive scripts simulating advanced persistent threats (APTs).
  • Need for realistic insider threat simulations in security testing.
Objectives:
  • Develop USB payload for automated attack sequence initiation.
  • Implement defense evasion via DLL Proxying to bypass Windows Defender.
  • Execute UAC bypass for privilege escalation.
  • Establish persistence and deploy C2 agent for remote access.
  • Exfiltrate credentials by bypassing App-Bound Encryption (ABE).

3. Background Study

USB Rubber Ducky attacks exploit the trust operating systems place in keyboard devices. Research shows these attacks can type commands much faster than humans, making them hard to detect with traditional antivirus software that looks for known malware patterns.

Current Limitations: Existing public Rubber Ducky scripts are easily caught by modern Windows Defender. They also fail to demonstrate complete attack scenarios from initial access through to data theft.

This Project's Solution: Combines multiple advanced techniques (DLL side-loading, UAC bypass, credential stealing) into one automated payload that successfully evades Windows 11 security defenses.

4. Design (System Architecture)

Attack Flow:

  1. USB insertion triggers Ducky Script keystroke injection
  2. PowerShell dropper downloads malicious DLL + legitimate binary
  3. DLL Proxying executes encrypted Sliver shellcode
  4. Process injection into Edge browser for stealth
  5. C2 callback establishes remote access channel
  6. Post-exploitation: UAC bypass, persistence, credential theft
